## arXiv:2603.02206v2

https://arxiv.org/html/2603.02206v2
```
VoiceAgentRAG: Solving the RAG Latency Bottleneck in Real-Time Voice Agents Using Dual-Agent Architectures
核心Idea是 Slow Thinker: Predictive Prefetching。 
本质是 foreground fast talker + background slow thinker，把 RAG 预取到 语义缓存，解决 voice agent 里“检索导致沉默”的问题。**Speculative Acceleration** 策略的一种，这种策略在很多环节中都可以实验性应用，latency-aware很好的措施。  latency（说话不能卡）是重要特性。 
```

```text
                         ┌────────────────────────────┐
conversation stream ────>│ Slow Thinker (background)  │
                         │ - 预测后续话题              │
                         │ - 提前检索知识              │
                         │ - 写入 semantic语义 cache       │
                         └─────────────┬──────────────┘
                                       │
                                       ▼
                              semantic cache
                                       ▲
                                       │ sub-ms hit
                         ┌─────────────┴──────────────┐
user asks now ──────────>│ Fast Talker (foreground)    │
                         │ - 不直接查大库              │
                         │ - 只读缓存                  │
                         │ - 立即组织语音回答          │
                         └────────────────────────────┘
```
```
Slow 策略:用 GPT-4o-mini（temp=0.3）滑窗最近 6 轮预测 3-5 个后续话题，并行asyncio.gather 预取。

Fast Talker 不光"只读缓存"——它没命中，也会顺手把结果写回缓存。

*不要等用户问完才检索；让后台慢 agent 持续预测下一步可能需要的知识，前台快 agent 只读亚毫秒缓存**。
Slow Thinker 不直接说话，只维护 `hot_context_cache`。
对工具等待超过 300ms 的场景，配合 OpenAI Realtime preamble：“我帮你查一下订单政策。”
Retrieval speedup 316X 提升，但仍可能被llm 生成盖住， 能把检索这一环的"沉默风险"消掉。 
```
## arXiv:2507.23159
https://arxiv.org/html/2507.23159v4
```
Full-Duplex-Bench v1.5: Evaluating Overlap Handling for Full-Duplex Speech Models
本质研究 **当用户和模型同时说话时，模型知道该停、该继续、该忽略、还是该回应吗？**

本质是说 全双工系统承诺可以“边听边说”，真实对话里会出现四类 overlap 需要不同的打断/turn-detect策略，更智能。 turn-taking（什么时候该接话）。
```
```
| user interruption | 用户打断：“等等，不是这个地址” | 立刻停止或让出 |
| listener backchannel | 用户：“嗯嗯”“对” | 不要误以为是新问题 |
| side conversation | 用户旁边有人说话 | 忽略非定向语音 |
| ambient speech | 背景广播/电视 | 稳定保持任务 |
场景中需要chaos方式注入干扰。
```
```text
scripted audio scenario
      │
      ▼
model under test speaks/listens
      │
      ├── behavior classification
      ├── stop latency
      ├── response latency
      ├── prosody adaptation
      └── perceived被感知的 speech quality
```
