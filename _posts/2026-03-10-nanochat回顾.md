# LLM 训练本质
```
LLM 训练过程把互联网上万亿 token 的文本**压缩**成了几十亿参数的**非线性函数**。这个函数的本质是：给定前文，输出下一个 token 的概率分布。
所有知识 纠缠在同一组高维向量空间的非线性映射里。多层非线性组合使其成为强大的函数逼近器。
训练数据约 10TB 文本，模型参数约 70B × 2 bytes ≈ 140GB。**压缩比约 70-130 倍**. 本质是压缩数据分布（distribution），对语言分布的参数化建模，而不是对文本本身的压缩。


概率空间是**确定的**（函数固定），但采样路径**不确定**（temperature > 0 时）。第一个 token 一旦选定，后续所有条件概率空间被约束——就像走迷宫，每个岔口选了一条路，后面的可选路径就变了。**未来在概率意义上被第一个 token 压缩了，但不是被决定了。** 每一步都在更新条件空间。

LLM 的参数 ≈ 把人类文本世界的统计规律压缩成一个概率函数，这个函数是世界规律的有损近似，训练 = 数据自动写程序，推理 = 运行这个程序。 
```

## 参数高效微调 (PEFT)
```
LoRA
 核心思想: 冻结原始权重 W，旁边加一个低秩分解矩阵 ΔW = B·A
 y = x · W          W: (d_in, d_out) 如 (4096, 4096)
 
 LoRA 改造后:
    y = x · W + x · B · A
        ↑ 冻结    ↑ 可训练
                  B: (d_in, r)    如 (4096, 16)
                  A: (r, d_out)   如 (16, 4096)
 r = LoRA 的秩 (rank)，通常 8~64，决定 ΔW 的表达能力

 Transformer 中的线性层:
    │ Attention:                                    │
    │   W_q  ← LoRA ✓  (查询投影)                  │
    │   W_k  ← LoRA ✓  (键投影)                    │
    │   W_v  ← LoRA ✓  (值投影)                    │
    │   W_o  ← LoRA ✓  (输出投影)                  │
    │                                              │
    │ MLP:                                          │
    │   W_gate ← LoRA ✓  (SwiGLU 门控)             │
    │   W_up   ← LoRA ✓  (扩展)                    │
    │   W_down ← LoRA ✓  (压缩)                    │
    │                                              │
    │ 通常: 至少加在 Q 和 V 上效果就不错            │
```

## Head 思路
```
MHA (Multi-Head Attention) — 标准多头注意力
  每个 Query 头都有自己的 K 头和 V 头

MQA (Multi-Query Attention) — 多查询注意力
所有 Q 头共享 1 个 K 头和 1 个 V 头

GQA (Grouped Query Attention) — 分组查询注意力
 折中: 把 Q 头分成几组，每组共享一个 KV 头

MLA (Multi-head Latent Attention) — 多头潜在注意力
   本质将K V 降维，减少kv cache，能还原。 
    c = x · W_DKV                    (降维: d_model → d_latent)
    K = c · W_UK                     (升维: d_latent → n_head × head_dim)
    V = c · W_UV

```

## RL 

RL 本质：生成回答 → 打分 → 强化高分回答的生成概率。
HuggingFace TRL (Transformer Reinforcement Learning)可以比较方便进行。 
GRPO = Group Relative Policy Optimization， 数据集DeepMath-103K 只有问题和标准答案，没有偏好对，没有人工标注打分。
每个问题生成一组（G 个）回答，用奖励函数/强llm打分，算相对优势，advantage = +1.0 推高token的概率。 
重复 → 模型越来越倾向于生成高分回答。 本质 在条件概率分布上做偏移（distribution shift），让概率质量向“高 reward 的输出子空间坍缩”。
