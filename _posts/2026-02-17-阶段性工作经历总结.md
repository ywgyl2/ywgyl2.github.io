# 近年来的工作

近年来 一直负责业务线系统体系的整体技术规划和演进。着眼于把一个"能跑"的多系统业务体系，体系化建设为**稳定、性能可控、成本可控**的体系。 
    
- **高可用**：推进 Active-Active 架构落地，多layer failover/autofailover, 完善多服务成熟度治理体系（Service Maturity）。
- **高性能优化**：体系化解决业务线的性能盲区和隐患。从 Nginx / Linux 内核参数调优，到 Java 全链路组件治理（连接池、消息队列、缓存、序列化、日志框架等，阅读源码troubleshooting），到多链路性能风险治理、架构与容量标准化。典型治理项包括：多组件使用治理(Jedis/Druid/HikariCp...)、EventBus 同步锁优化、Kafka Producer 阻塞修复、Tomcat扩展、缓存方案优化（延迟双删,KeyLock,localcache）、Threadlocal标准化，日志框架同步瓶颈解决、Protobuf 序列化升级提升吞吐、内存分配器替换降低内存占用、cpu/mem 高risk issues、核心链路 性能基线标准定义和达标 等等。
- **容量管理和成本治理Cost-ware**：主导容量与成本管理，推动所有集群的 部署架构优化 和 标准化，建立清晰的容量标准，实施成本优化，实现**300w$+**的成本节省。
- **可观测性**：监控告警体系从零开始体系化构建，基于开源/自研 可观测性体系。
- **标准化与组织机制**：定义标准化 和 技术事项协同的 组织机制，完成 业务线 压测性能基线/容量/架构 标准化，培养团队专项小组，体系化沉淀能力。

之前经历了一家头部互联网公司（电商方向），作为技术主管，负责 2 条业务线的业务技术规划、演进，与业务、产品总监一起参与电商供应链体系的 Roadmap 演进。

- 重构定义仓储、库存中心技术系统，通过服务化和重构解决系统的容量瓶颈、关键缺陷和研发效率问题。
- 运用数据技术大幅提升数据准确率，解决十亿级别库存财务账务一致性问题。
- 规划调整仓网结构分布、多供应商策略，大幅提升履约投递时效、降低履约成本。

再之前经历了 2 家电商 Startup 公司：
